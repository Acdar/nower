前言

Mower入门指北，下文简称本指北。

Mower开发组制作的Mower是一个开源的Arknights（明日方舟）模拟器助手，旨在帮助玩家更高效地管理游戏内的干员和资源。Mower提供了多种功能，包括自动排班、资源管理、邮件通知等，极大地提升了游戏体验。

本指北将介绍Mower的下载、运行、更新方法，以及如何配置Mower的各项设置，帮助用户快速上手使用Mower。

本指北的内容将持续更新，欢迎用户提出建议和反馈。

本指北可用Chrome、Firfox、Safari等主流浏览器直接打开。

若需了解更多Mower相关问题，可加入QQ群Mower用户交流群或QQ频道ArkMower，文明交流互助。

下载、运行及更新

Windows

下载和运行

  1. 在群文件"Mower下载"文件夹中获取最新全量包
  2. 在英文路径下创建Mower工作文件夹
  3. 将全量包内容解压至此
  4. 运行"mower.exe"启动Mower

手动更新

  1. 在群文件"Mower下载"文件夹中获取最新增量包
  2. 将增量包内容解压至Mower工作文件夹，覆盖原有文件
  3. 运行"mower.exe"启动Mower

自动更新

  1. 从群文件"Mower下载"获取最新下载器
  2. 解压并运行"MowerDownload.exe"，选择安装路径
  3. 选择目标版本
  4. 选择更新类型：
    • 仅更新新干员资源（保留当前版本）
    • 更新Mower代码（体验新功能）
    • 全量更新（干员资源+代码更新）
    • 替换新版本DLL（更新MAA调用文件）
  5. 等待更新完成，关闭更新提示窗口

MacOS、GNU/Linux

MacOS、GNU/Linux使用源码运行，本指北以MacOS示例

首次运行

  1. 配置环境

    访问MAA项目地址：https://github.com/MaaAssistantArknights/MaaAssistantArknights

    安装mac版MAA，并下载Windows版MAA，复制其python文件夹到

    /Applications/MAA.app/Contents/Resources

    将/Applications/MAA.app/Contents/Frameworks里的内容复制到

    /Applications/MAA.app/Contents/Resources

    安装蓝叠模拟器Air版：https://www.bluestacks.com/mac

    安装MuMu模拟器Mac版：https://mumu.163.com/

    模拟器设置

    • 分辨率：
      1920x1080
    • 每英寸点数（DPI）：
      280
    • 启用Android调试(ADB)

    安装Mac软件包管理器和Mower运行依赖 

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install git git-lfs python@3.12 android-platform-tools zbar

  2. 构建Mower

    （假设你已设置好网络代理）在终端中执行以下命令：

    #创建mower工作目录
cd
mkdir ~/ark
cd ~/ark
#克隆mower源代码
git lfs install
git config --global lfs.concurrenttransfers 50
git clone -c https://github.com/ArkMowers/arknights-mower.git --branch 2025.6.1
cd arknights-mower
#构建python虚拟环境并安装依赖
python3.12 -m venv venv
source venv/bin/activate
pip install -r requirements.in
cd ui
npm install
npm run build
  3. 运行Mower

    通过终端启动：

    cd ~/ark/arknights-mower
source venv/bin/activate
python3 webview_ui.py
  4. 配置Mower


    ADB路径：

    adb
    ADB连接地址：
    127.0.0.1:5555
    模拟器类型：
    其它
    启动游戏方式：
    使用adb命令启动
    截图方案：
    DroidCast
    MAA目录：
    /Applications/MAA.app/Contents/Resources
    连接配置：
    General

  5. 停止Mower

    Mower运行时关闭终端

源码更新

（假设你已设置好网络代理）在终端中执行以下命令：

#进入mower工作目录
cd ~/ark/arknights-mower
#同步mower最新源代码
git fetch origin 2025.6.1 --progress
git switch -f 2025.6.1 --progress
git reset --hard origin/2025.6.1
#激活python虚拟环境并更新mower运行依赖
source venv/bin/activate
pip install -r requirements.in
cd ui
npm install
npm run build

Mower设置

基础设置

根据个人需求配置基本参数

宿舍黑名单

设置永不进宿舍的干员

示例：零心情工作干员Lancet-2

邮件提醒

配置接收任务通知的邮箱，通过问号图标查看授权码获取方式。通知等级分为：

  • INFO：基本信息
  • WARNING：警告信息
  • ERROR：错误信息
提示：高级别包含低级别信息，时差修正单位为小时。清空收件人字段可将邮件发送至本人邮箱

森空岛相关

输入账号密码后，Mower可自动完成森空岛每日签到，并为仓库物品混合读取和无缝合成材料提供支持。

无缝合成材料设置

位置：全部设置 → Mower设置 → 基建设置

功能特点：

  • 由"宿舍不养闲人回满"功能触发
  • 需启用森空岛仓库读取功能
  • 当目标材料数量低于上限且子材料数量高于下限时自动合成
  • 支持多种材料合成策略

特殊说明：

  • 九色鹿垫刀材料：消耗4心情以下的材料
  • 九色鹿暴击材料：消耗4心情的材料
  • 家具零件需用"家具零件_碳"格式区分配方
推荐合成路径：碳素 → 碳素组 → 家具零件_碳素组 → 材料
提示：合成截图保存在screenshot/workshop文件夹，条件不满足时发送邮件通知

作业导入

从群文件或频道下载图片/JSON格式作业文件，通过"排班编辑→导入排班"加载

注意：作业文件不包含主表黑名单列表，具体限制需参考作业说明或咨询作者。

葛朗台跑单

跑单机制：但书、龙舌兰等干员的技能效果仅在订单生成瞬间生效。在贸易站安排高效干员，在订单即将结束时换上特殊干员，新订单生成后换回原班组的操作。

配置方法：将跑单干员置于贸易站替换组首位

选项说明：

  • 启用葛朗台跑单：Mower会在跑单前数分钟（前置延时）准备干员，并在新单生成前数秒（缓冲时间）放入贸易站
  • 禁用葛朗台跑单：干员立即放入贸易站并使用无人机加速
提示：网络或配置不佳时建议延长前置时间。

心情阈值

干员或班组平均心情达到此百分比时，Mower会在高优位置充足时安排休息。实际心情值计算公式：

实际心情值 = 心情下限 + 心情阈值 × (心情上限 - 心情下限)
示例：阈值设为65%，夕在感知模式下的实际心情值为12 + 0.65 × 12 = 19.8

急救阈值

常规流程：干员到达心情阈值时若高优位充足则正常休息。

急救模式：当整组心情低于"心情阈值×急救阈值"且宿舍有空位时，强制安排休息。

设为0时禁用急救模式。

宿舍不养闲人

自动移出已回满的干员，替换为待回复干员，保持宿舍满员以触发依赖型技能（如黑键、乌有、迷迭香）。启用后会增加任务频率。

肥鸭防呆与肥鸭阈值

  • 开启：所充干员均大于90%则肥鸭待机30分钟
  • 关闭：所充干员均大于肥鸭阈值则优先充能心情最低者

宿舍优先级排序

某些布局中上层非VIP位回复能力可能高于下层VIP位，与默认逻辑冲突时可手动设置优先级顺序。更换宿舍布局后需更新此设置。

排班教学

页面介绍

排班编辑界面分为四个功能区域：

  1. 条件区：设置触发条件
  2. 副表任务区：配置执行任务
  3. 设施区：显示工作站布局
  4. 设置区：干员管理

操作指南：

  • 主表的条件及任务区需通过"新建副表"激活
  • 拖拽调整设施区位置需与实际游戏布局一致
  • 输入干员名称时可使用首字母缩写快速匹配

主力与替换

主班（高效组）：核心高效干员，优先安排工作

替班：替补干员，在主班休息时自动轮换

轮换规则：

  • Mower选择列表中靠前且未被占用的干员接替工作
  • 一名干员可担任多名主力的替换
  • 主力选择原则：维持长工作时间高效率状态

验证规则：

  • 高效组不可使用龙舌兰、但书（仅限替换组）
  • 同一替换组不可同时安排龙舌兰和但书
  • 替换组不可包含高效组干员
  • 确保替换组数量充足

班组管理

填写相同组名可将干员绑定为班组：

  • 同组干员一起上下班
  • 不同工作站干员也可绑定
  • 宿管不可绑组

小组：无中枢的班组

大组：包含中枢的班组

宿舍管理

配置要求：

  • 每个宿舍安排一名单回宿管和一名群回宿管
  • 下层宿舍回复能力不超过上层宿舍
  • 单回宿管数值要求：琴柳(0.7)≥蜜莓(0.7)≥车尔尼(0.65)≥波登可(0.65)
  • 群回宿管数值要求：流明(0.25/0.4)≥杜林(0.25)≥桃金娘(0.15)≥波登可(0.15)

低优先级设置

仅作用于主班干员，配置后：

  • 不会主动前往高优位休息
  • 高优位充足时可使用
  • 高优干员需入驻时会被移出

推荐适用情况：

  • 非绑组的会客室/办公室等低收益站干员
  • 回复能力显著高于同组成员的干员
  • 主副班效率相近的干员

工作站基础消耗：

  • 贸易站/制造站：无减免时为1
  • 入驻2个干员：0.05减免
  • 入驻3个干员：0.1减免

中枢效果：每入驻1个干员全局-0.05消耗（上限0.25）

工休比计算

工作时间占比公式：

工休比 = 回复效率 / (回复效率 + 消耗效率)

肥鸭充能情况：

工休比 = (回复效率 + 肥鸭提供效率) / (回复效率 + 消耗效率)

应用场景：

  • 计算布局收益
  • 评估大组轮休可行性
  • 确定替换组能替多少人
  • 计算肥鸭充能收益

常见问题

连接相关

Q：如何设置模拟器路径和连接地址？

A：Mower设置中填写ADB路径和连接地址：

  • ADB路径格式：[模拟器地址\adb.exe]
  • 连接地址格式：127.0.0.1:端口号

常见模拟器默认端口：

模拟器默认端口
蓝叠模拟器5555
夜神模拟器62001
MuMu模拟器/MuMuX7555
MuMu12模拟器16384
MuMu模拟器Mac版5555
逍遥模拟器21503
雷电模拟器5555 或 emulator-5554

副表相关

Q：为什么心情副表在干员到达阈值时未生效？

A：可能原因：

  1. Mower心情读取间隔：歌蒂、见行者每0.5小时读取一次，其他干员2.5小时读取一次
  2. 干员入驻宿舍后只读一次剩余时间
  3. 触发时机设置不当
解决方案：给宿管写用尽，给跑单干员写用尽重读

Q：副表提示排班表验证错误怎么办？

A：干员在主表或其他副表的其他位置有工作安排，需要在副表对应位置写死其他干员。

用户需知

开源性质

Mower是开源、免费工具，无任何强制收费项目。请勿相信任何收费版本或盗版软件。

社区相关

合理使用群组Mower用户交流群和频道ArkMower资源，文明交流互助。

更新频率

保持程序更新，及时获取最新功能和修复。可通过QQ群或GitHub获取最新版本。

数据安全

定期备份配置文件，防止意外丢失设置数据。

手动操作

合理安排任务时间，避免在游戏期间执行自动化操作。

长时挂机

长时间离开时确保设备电源充足并保持网络连接稳定。

异常处理

若遇到异常情况，及时查看日志文件排查问题。

多设备登录

不要在多个设备同时登录同一账号，避免账号安全风险。

反馈建议

发现程序问题及时反馈，提供详细的重现步骤和日志。

模拟器设置

确保模拟器设置正确，分辨率为1920x1080，DPI设为280。

隐私安全

小心网络钓鱼，避免泄露个人隐私信息。

游戏性平衡

合理使用Mower，避免过度依赖自动化工具，保持游戏娱乐性。

寸止必要性

熟悉程序停止快捷键，遇到问题可立即中断任务。

脚本工具论

Mower只是一个脚本工具，当它与你的快乐冲突时，请拥抱自己的生活。
